JUL 2026

Adding dark mode

Added a site-wide dark mode with a toggle in the nav. Since every color on the site already went through CSS variables, the whole theme flips by just redefining those variables under a second attribute selector and toggling one attribute on the page — no per-component dark styles needed. Remembers your choice across visits, and defaults to your system preference the first time. Along the way, found two colors that had been hardcoded instead of using variables — a good reminder of why that's worth avoiding.

Chasing a phantom cache bug (that wasn't)

Spent a long session convinced the hero banner not showing on four pages was a browser caching issue — cleared cache, tried incognito, disabled cache in DevTools, none of it worked. Turned out to be a real bug: a CSS url() path resolves relative to the CSS file's own location, not the HTML page that loads it, so the same relative path worked from index.html but broke from inside css/styles.css. Also found the portfolio was actually being served from a different port than I was testing against the whole time. Lesson: rule things out systematically, and check what URL is actually being requested before assuming it's a cache problem.
